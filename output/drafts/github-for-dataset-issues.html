Using GitHub to track dataset issues and more

March 06, 2014
By Peter Desmet
    Tagged:
  • data publication
  • GitHub

Part of our mission in building an open terrestrial and freshwater observatory is the rapid publication1 of the biodiversity data it generates. In addition, we unlock data that have been collected and managed by our institute over the last two decades. The bulk of these data are occurrence data, which we publish as open, standardized, well-documented, GBIF-registered Darwin Core Archives on our GBIF Integrated Publishing Toolkit (IPT).

Open issue tracking for data

Even though we rigorously review our (to be) published data with tools such as Open Refine, there will always be errors and inconsistencies in our data. Inspired by the open issue tracking by Canadensys for the Database of Vascular Plants of Canada (VASCAN), the OKFN Labs for public datasets, and VertNet for specimen data, we have started to do the same.

Issues

We have also started to publicly document and solve issues with the data and metadata in a GitHub repository (https://github.com/LifeWatchINBO/vis-inland-occurrences/issues) as well as providing the metadata as an easy to read text file (https://github.com/LifeWatchINBO/vis-inland-occurrences/blob/master/paper.md). This not only allows users to be informed of ongoing activities, but also allows them to records issues and contribute suggestions themselves. We hope this creates an open community of data users.
